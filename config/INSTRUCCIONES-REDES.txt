Redes sociales y ranking (Meta)

IMPORTANTE: usa el servidor del proyecto para OAuth y para el ranking.

  npm start
  Abre http://localhost:3333/ (puerto = PORT o 3333)

No uses solo `npx serve` u otro servidor estático: no existen las rutas /api ni OAuth.

────────────────────────────────────────
1) App en Meta (developers.facebook.com)
────────────────────────────────────────
  • https://developers.facebook.com → Crear app.

  • Producto "Inicio de sesión con Facebook":
    Configuración → URI de redirección de OAuth válidas (exactas):
      http://localhost:3333/oauth/meta/callback
    Repite con tu puerto si cambias PORT. Si entras con 127.0.0.1, añade también:
      http://127.0.0.1:3333/oauth/meta/callback
    Para el botón «Abrir Facebook» (Share Dialog), añade también (mismo puerto si aplica):
      http://localhost:3333/comunidad.html
      http://127.0.0.1:3333/comunidad.html
    En producción, añade la misma ruta con tu dominio HTTPS (ej. https://tudominio.com/comunidad.html).

  • Copia App ID y App Secret (Configuración de la app → Básico).

  • Permisos: para email y nombre se usan email y public_profile en el diálogo OAuth.

  • Modo desarrollo: solo cuentas de prueba / administradores pueden autorizar hasta pasar revisión y publicar la app.

────────────────────────────────────────
2) Credenciales locales
────────────────────────────────────────
  • Copia redes-secrets.example.json → redes-secrets.json en la raíz del proyecto.

  • Edita:
      { "metaAppId": "TU_APP_ID", "metaAppSecret": "TU_SECRET" }

  • redes-secrets.json no debe subirse a git (está en .gitignore).

────────────────────────────────────────
3) SDK JavaScript de Meta en Redes sociales
────────────────────────────────────────
  • La página carga el SDK (connect.facebook.net) y usa FB.init con el mismo App ID que redes-secrets.json
    (el navegador lo obtiene de GET /api/redes/meta-app-id; no pongas el App Secret en el cliente).

  • En la app Meta, en Configuración → Básico:
    - URL del sitio: http://localhost:3333/ (ajusta puerto si usas PORT)

  • Botones Facebook / Instagram: FB.login con permisos email y public_profile; el accessToken se envía a
    POST /api/redes/verify-sdk-token. El servidor valida el token con Graph debug_token y luego llama a
    /me?fields=id,name,email. El usuario queda guardado por id de Graph API (campo "id").

  • Instagram (consumidor): suele usarse el mismo inicio con Facebook; el segundo botón marca el canal
    "instagram" en el servidor para exigir ambas vinculaciones en el ranking.

────────────────────────────────────────
4) Ranking (Puntuaciones)
────────────────────────────────────────
  • En puntuaciones aparecen jugadores que han vinculado Facebook en esta app (Instagram sigue opcional para marcar el canal en redes).

  • Cada victoria suma 1 si ganaste la partida:
      - Batalla (astro-sync): último astronauta en pie.
      - Modo zombie cooperativo: oleadas completadas con victoria.

  • Los datos se guardan en el servidor en data/scores-meta.json (generado automáticamente; en .gitignore).

────────────────────────────────────────
5) Consumir más datos de Meta (opcional)
────────────────────────────────────────
  • Facebook / páginas: Graph API según permisos y revisión de la app.
  • Instagram profesional: Instagram Graph API (cuenta Business/Creator + página de Facebook).

  Documentación: https://developers.facebook.com/docs/graph-api

────────────────────────────────────────
6) Comunidad: compartir en el muro del jugador
────────────────────────────────────────
  • La página comunidad.html debe incluir <div id="fb-root"></div> al inicio del body; sin eso el
    diálogo FB.ui (Compartir) puede fallar.
  • En localhost / 127.0.0.1 se usa el Share Dialog por URL (facebook.com/dialog/share?app_id=…&href=…
    &quote=…&redirect_uri=…), no sharer.php: Facebook redirigía a share_channel/# sin parámetros.
  • El botón «Abrir Facebook» comparte la URL /share/comunidad/ID: el servidor genera HTML con meta
    Open Graph (título, texto e imagen leídos de MySQL) para que el cuadro de Facebook muestre la vista previa.
  • Tras publicar, el jugador usa «Abrir Facebook» en su tarjeta (clic directo). Script SQL:
    sql/alter-publicaciones-facebook-enlace.sql si la base ya existía.
  • No es posible publicar en el muro personal sin acción del usuario desde una app genérica; en
    Página de Facebook haría falta Graph API con permisos de página y revisión de la app.
